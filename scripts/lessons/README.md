# Видео-уроки: пайплайн создания

Папка хранит **исходники** всех анимированных видео-уроков сайта `nish-prep`.

## ⭐ Текущий стандарт (v2 — 2026-04-20)

Каждая сцена видео:
1. **Текст появляется** на экране
2. **2 секунды тишины** — ребёнок читает
3. **Аудио объяснения** (TTS казахский Aigul, rate -10%) — играет полностью
4. **2 секунды тишины** — ребёнок усваивает
5. **Переход** на следующую сцену

Между двумя сценами в итоге **4 секунды паузы** (2с после + 2с до).

Реализация: каждая сцена генерится **отдельным TTS-запросом**, склеиваются через ffmpeg с silence-mp3 файлами. Это устраняет проблему рассинхрона из-за того что edge-tts может разбивать предложения иначе чем мы ожидаем.

**Длительность сцены в `src/data/lessons.tsx`** = `PRE_MS (2000) + audio_duration + POST_MS (2000)` — берётся из `timings.json` после генерации.

## Структура

```
scripts/lessons/
├── generate_all.py      ← скрипт генерации TTS + извлечения таймингов
├── texts/               ← текстовые сценарии на казахском (для редактирования)
│   ├── lesson-01-natural-numbers.txt
│   ├── lesson-02-razryad.txt
│   ├── ...
└── README.md            ← этот файл
```

Готовые аудио лежат в `public/lessons/lesson-XX-audio.mp3`.
Анимированные сцены — в `src/data/lessons.tsx`.

## Полный пайплайн создания нового урока

### 1. Написать сценарий на казахском

Создай файл `texts/lesson-XX-slug.txt` и раскидай текст по сценам с разделителем `═══`.
Каждая сцена — 1-5 коротких предложений. Короткие сцены лучше — дети не устают.

Длительность целевая: **50-80 секунд** на урок.

### 2. Добавить сценарий в `generate_all.py`

В словарь `LESSONS` добавь запись:

```python
"lesson-XX-audio.mp3": [
    # Scene 1
    ["Предложение один.", "Предложение два."],
    # Scene 2
    ["Предложение три.", ...],
    # ...
],
```

**Важно:** каждая сцена = список предложений (строк). Точка/восклицание/вопрос = конец предложения.
edge-tts выдаёт таймкод начала каждого предложения → из них считаются длительности сцен.

### 3. Запустить генерацию

```bash
cd scripts/lessons
python generate_all.py
```

Скрипт:
- Генерирует TTS через Microsoft Edge (голос `kk-KZ-AigulNeural`)
- Для каждого урока создаёт `lesson-XX-audio.mp3`
- В `timings.json` пишет длительности сцен в миллисекундах
- Распечатывает: `sentences: N, scene durations: [...]`

### 4. Скопировать аудио в public

```bash
cp scripts/lessons/lesson-XX-audio.mp3 public/lessons/
```

(В идеале изменить скрипт чтобы сразу писал в public — сейчас он пишет в текущую папку.)

### 5. Добавить сцены в `src/data/lessons.tsx`

a) Расширить тип `LessonKey`:
```typescript
export type LessonKey =
  | "lesson-01-natural-numbers"
  | ...
  | "lesson-XX-slug";
```

b) Добавить `LessonConfig` с `scenes: Scene[]`. Каждая сцена:
```typescript
{
  duration: <ms из timings.json>,
  render: () => (<div>JSX сцены</div>),
}
```

Длительности **ОБЯЗАТЕЛЬНО** брать из `timings.json` — иначе аудио и картинка разойдутся.

c) Добавить `matchSection` regex — по какому заголовку теории видео встраивается:
```typescript
matchSection: /разряд/i,  // встроится в раздел со словом "Разряд"
```

d) Добавить в `LESSONS_BY_KEY`:
```typescript
export const LESSONS_BY_KEY: Record<LessonKey, LessonConfig> = {
  ...,
  "lesson-XX-slug": LESSON_XX_CONFIG,
};
```

### 6. Подключить на странице темы

В `src/app/subjects/[slug]/[topicId]/theory/page.tsx`:

```typescript
<TheoryRenderer
  ...
  sectionLessonKeys={
    topic.order === 1 && topic.subject.slug === "math"
      ? (["lesson-02-razryad", ..., "lesson-XX-slug"] as LessonKey[])
      : []
  }
/>
```

Интро-видео (первое, над теорией) подключается через `<LessonVideoByKey lessonKey="lesson-XX" />` отдельно.

### 7. Задеплоить

```bash
git add public/lessons/lesson-XX-audio.mp3 src/data/lessons.tsx src/app/subjects/[slug]/[topicId]/theory/page.tsx scripts/lessons/
git commit -m "Add lesson XX: <tема>"
npx vercel --prod --yes
```

## Правила дизайна сцен

- **Aspect-video** (16:9) — содержимое не должно обрезаться на мобилке
- Большой текст: `text-3xl md:text-6xl` для заголовков, `text-xl md:text-3xl` для примеров
- Font-mono для чисел и формул
- Цвета по смыслу:
  - `text-emerald-400` — правильные/положительные ответы
  - `text-red-400` — ошибки/предупреждения
  - `text-amber-300` — важное/выделенное
  - `text-sky-400` / `text-indigo-400` — акцент
- Анимации: `animate-slide-in` (слева), `animate-fade-up` (снизу)
- `animationDelay` относительно НАЧАЛА сцены — считай от 0, по мере произношения
- Не перебарщивать с контентом на сцене: max 4-5 элементов на экран

## Правила сценария

- Тон: дружелюбный учитель для 5-6 класса
- Сложные слова — разбить на короткие предложения
- Примеры: простые числа (до 20 для сложения/вычитания, до 12×12 для умножения)
- Структура каждого урока:
  1. Заставка с названием темы
  2. Определение или ключевая идея
  3-N. Примеры / разбор с визуализацией
  N-1. Типичная ошибка / важное замечание
  N. Финальный призыв (короткий, 1 предложение)

## Отладка

- **Аудио обгоняет картинку** → сверь `duration` в lessons.tsx с `timings.json`
- **Нет звука в браузере** → автоплей заблокирован, нужен клик на "БАСТАУ" (уже реализовано)
- **Сцены обрезаются на мобилке** → уменьши `text-XL` и `gap-N` значения на breakpoint без `md:`
- **Видео не появляется на странице** → проверь `matchSection` regex против заголовка в теории

## Голоса edge-tts

Для казахского доступно 2 голоса:
- `kk-KZ-AigulNeural` — женский, дружелюбный ⭐ (используем по умолчанию)
- `kk-KZ-DauletNeural` — мужской, тоже дружелюбный

Rate `-5%` чуть медленнее нормы — комфортно для детей.

## Зависимости

Python 3.10+ с пакетами:
- `edge-tts`
- `mutagen` (опционально, для проверки длительности)

Установка:
```bash
pip install edge-tts mutagen
```
